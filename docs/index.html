場景一：用戶洞察自動整合

用戶研究

「不再手動整理，讓 AI 先篩一遍」

  • 全渠道掃描： AI 每日彙整 App Store 評論、Intercom 對話、問卷回饋，自動分群與情緒標記。
  • 洞察萃取： 將百筆回饋濃縮為 3–5 個可行動主題，附原始引言作為佐證。
  • 定期推播： 每週一早晨自動推送摘要至 Slack，PM 直接進入決策，不再花時間整理。

場景二：PRD 智慧起草與一致性審查

NOTION · PRD 草稿 AI 生成中…

FEATURE

一鍵重新訂購：讓用戶快速複製歷史訂單

USER STORY

身為回購用戶，我希望可以在訂單頁一鍵複製上次的購物清單，這樣我就不需要重新搜尋每個商品。

ACCEPTANCE CRITERIA

✓ 訂單頁顯示「再買一次」按鈕

✓ 點擊後商品加入購物車，缺貨品項標示提示

✓ 支援最近 6 個月訂單

⚠ AI 衝突偵測

與 Sprint 22「購物車重構」存在依賴關係，建議確認排期。

需求文件

「描述目標，AI 幫你起草第一版」

  • 一句話起草： PM 輸入功能描述，AI 生成包含用戶故事、驗收標準與邊界條件的 PRD 草稿。
  • 一致性審查： AI 自動比對現有 Roadmap，標記潛在衝突與重複需求，避免遺漏。
  • 工程對齊： 草稿同步至 Notion，AI 標記技術不確定點，供工程師直接評估。

場景三：產品指標異常偵測與 SOP 引導

產品監控

「指標下跌的那一刻，AI 已在診斷」

  • 即時偵測： 監控轉換率、留存率、錯誤率等關鍵指標，偏差超過閾值立即觸發警報。
  • 自動根因分析： AI 比對近期版本變更與用戶行為路徑，定位最可能的問題來源。
  • SOP 引導處置： 依警報類型展開標準應對流程，引導 PM 逐步確認並做出決策。

PM 的行為改變框架

Stop 停止

  • 手動整理用戶訪談與回饋
  • 從零開始撰寫 PRD 草稿
  • 週五下午複製數據做週報
  • 盯著 dashboard 等待異常

Start 開始

  • 審核 AI 整理的用戶洞察摘要
  • 精煉 AI 起草的需求文件
  • 設定指標警報閾值與應對 SOP
  • 委派可描述的重複任務給 Agent

Keep 保留

  • 親自進行用戶訪談
  • 優先級的最終判斷
  • 跨部門的信任與關係
  • 策略方向的定義與溝通

AI 如何串聯 PM 的工具版圖

Model Context Protocol

PM 的工具版圖分散在 5 個以上的平台。MCP 是讓 AI Agent 能夠統一存取這些數據的「通用連接器」。

結果： AI 不再只看單一工具的片段，而是擁有完整的上下文，才能給出真正有用的洞察與行動建議。

整理資料 / 撰寫文件 / 做報告 60%
用戶訪談 / 策略思考 40%

大量時間消耗在可被自動化的執行工作上。

AI Agent 介入後

審核 AI 輸出 / 決策確認 20%
用戶訪談 / 策略 / 跨部門影響力 80%

PM 的核心價值回到不可被取代的事情上。
